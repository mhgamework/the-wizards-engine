Engine

Contains ideas for adding to the engine.

 

 

Show the percentage of 16ms (60FPS) the frame is currently using. This is a much more interesting number.

When marking objects for automated testing, a correct visualization can be chosen depending on the type! This might make tests a whole lot easier: Meshes are automatically visualized, frame buffers are paused to check the correctness, normal objects can be tree visualized, or converted to text, matrix visualization

 Add a feature to render a full frame in steps of 30ms, in order to render large scenes without lod for previews.

 

Focus more on a component based design http://cowboyprogramming.com/2007/01/05/evolve-your-heirachy/ (eg cleanly list all components seperately)

 

Add a text face renderer (2 sided?)

Visualize objects tree in 3D by showing a 2D graph, with text panels containing extra information. Can add lod to the panels, where the text changes from big name to more detailed

Add automated test for hotloading! NEEDED becaus it always break

Add some form of automated test running, at least starting the tests is good.

Support running the engine without rendering window.

Try using a build server, eg TeamCity.

Allow DI construction on test fixtures, annotated with a special attribute (eg integration tests for the engine allow injecting engine stuff, integration tests for skymerchant allow injecting skymerchant stuff)
